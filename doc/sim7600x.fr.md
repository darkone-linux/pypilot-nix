# Waveshare SIM7600X — 4G & LTE Cat-1 HAT

Notes pour utiliser le HAT, comprendre son architecture USB/UART et packager les
extras éventuels. Module NixOS associé : `modules/hardware/sim7600x-hat.nix`
(option `services.navigation.hardware.hats.enableSim7600x`).

## Le matériel

HAT 40 broches autour d'un module SIMCOM **SIM7600X** (Cat-1). Deux chemins vers
le Pi, plus un port de debug :

- **Port micro-USB « USB »** (recommandé) : relié au Pi par un câble micro-USB.
  Enumère un composite SIMCOM (**VID `1e0e`**) :
  - `cdc-wdm0` + `wwan0` — interface data QMI (driver `qmi_wwan`).
  - `/dev/ttyUSB0` — DIAG / diagnostic.
  - `/dev/ttyUSB1` — **NMEA GPS**.
  - `/dev/ttyUSB2` — **commandes AT**.
  - `/dev/ttyUSB3` — PPP / modem.
  - `/dev/ttyUSB4` — audio.
- **UART header** (GPIO14/15) : un seul canal AT, débit limité. Le HAT y câble sa
  série quoi qu'il arrive ; le module réserve 14/15 pour la détection de conflit.
- **Port micro-USB « USB TO UART »** (CP2102) : debug AT depuis un PC, jumper sur
  `C`. À ne pas confondre avec le port « USB ».

> **Jumpers** : les deux du milieu sur **B** ; relier **PWR↔3V3** pour
> l'auto-allumage du modem dès la mise sous tension (sinon il faut piloter
> PWRKEY, cf. plus bas). Antenne LTE sur le connecteur **main**.

## Ce que fait le module

- Charge `option`, `usb_wwan`, `cdc_wdm`, `qmi_wwan`.
- Active **ModemManager** en standalone (`networking.modemmanager.enable`), donc
  sans NetworkManager : il cohabite avec le `wpa_supplicant` des hôtes (wlan0 vs
  wwan0, pas de conflit).
- Pose des symlinks udev stables : `/dev/ttySIM_gps` (if01) et `/dev/ttySIM_at`
  (if02).
- Installe `mmcli` (modemmanager), `qmicli`/`mbimcli` (libqmi/libmbim),
  `usb-modeswitch` et `minicom`.
- `connection.enable` (défaut `true`) : oneshot `sim7600x-connect` qui attend le
  modem puis `mmcli --simple-connect`. L'IP est ensuite prise par le client DHCP
  du système sur `wwan0`.
- `gps.enable` (défaut `false`) : oneshot `sim7600x-gps` qui active le GNSS
  (NMEA + raw) via l'API location de ModemManager.

```nix
services.navigation.hardware.hats.enableSim7600x = true;
services.navigation.sim7600xHat = {
  apn = "internet"; # vide = auto-détection
  gps.enable = true;
};
```

## Bring-up et debug sur le banc

```sh
# le modem est-il vu ?
mmcli -L
mmcli -m any

# état de la connexion data
mmcli -m any --simple-connect="apn=internet,ip-type=ipv4"
ip addr show wwan0

# GNSS
mmcli -m any --location-enable-gps-nmea
mmcli -m any --location-get

# AT brut (debug)
minicom -D /dev/ttySIM_at -b 115200
```

### Note QMI raw-ip / DHCP

Sur noyaux récents, `qmi_wwan` met `wwan0` en mode **raw-ip** : un client DHCP
classique peut rester muet. Deux remèdes :

- laisser ModemManager appliquer l'IP du bearer (il sait gérer raw-ip), **ou**
- en QMI manuel : `qmicli -d /dev/cdc-wdm0 --wds-start-network=...`, forcer
  `echo Y > /sys/class/net/wwan0/qmi/raw_ip` puis `udhcpc -i wwan0`.

Vérifier le mode négocié influe sur le fonctionnement du DHCP ; à valider au banc
(niveau 3).

### PWRKEY manuel

Sans le jumper PWR↔3V3, le modem démarre éteint. Les démos Waveshare pulsent un
GPIO (selon la révision, BCM **4** ou **6**) ~1 s pour l'allumer. Préférer le
jumper d'auto-allumage pour un usage embarqué sans intervention.

## Packager les extras (TODO)

Rien d'obligatoire — `qmi_wwan` + ModemManager suffisent. Pistes si besoin :

- **`pkgs/simcom-wwan`** : driver out-of-tree que SIMCom recommande pour le
  SIM7600 sur certains noyaux (alternative à `qmi_wwan`). Build kernel-module
  (`stdenv` + `kernel.dev`), à recompiler par version de noyau.
- **CLI `sim7600x`** : petit wrapper (writeShellApplication) autour des
  incantations `mmcli`/`qmicli` (connect, status, gps, raw-ip) pour éviter de les
  retaper. Suffit largement pour la plupart des usages, pas de driver à compiler.

## GPS dans la pile navigation

Le port NMEA (`/dev/ttySIM_gps` / ttyUSB1) peut alimenter `gpsd`/Signal K comme
n'importe quelle source série : le déclarer via `serialDevices`/`nav-discover`
plutôt qu'en dur. L'horloge peut aussi être disciplinée sur ce flux GPS.

## À faire / vérifier au banc

Points non testables sans matériel, à confirmer (niveau 3) :

- **DHCP sur `wwan0`** : le module compte sur le client DHCP du système une fois
  le bearer monté. Or `qmi_wwan` négocie souvent du **raw-ip**, où un DHCP
  classique reste muet (cf. note QMI raw-ip plus haut). Alternative si ça coince :
  faire le oneshot `sim7600x-connect` appliquer explicitement l'IP du bearer
  (`mmcli --output-json` → `ip addr add` / `ip route add` / DNS) au lieu de
  déléguer au DHCP.
- **Numéros d'interface udev** : les symlinks supposent if01 = NMEA, if02 = AT
  (composition QMI standard). Selon le firmware/le mode USB, la numérotation peut
  différer ; vérifier avec `udevadm info -q property -n /dev/ttyUSB*` et ajuster
  les règles.
- **APN auto-détecté** : `--simple-connect` sans APN dépend de la base
  ModemManager ; selon l'opérateur il faudra renseigner `sim7600xHat.apn`.
- **Mode USB** : confirmer que la HAT enumère bien en QMI (`qmi_wwan`) et pas en
  RNDIS/ECM ; sinon adapter les modules noyau.
