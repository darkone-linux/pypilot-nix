# XPT2046 — HAT écran tactile SPI

Notes pour utiliser le HAT, comprendre l'overlay device-tree et calibrer le
tactile. Module NixOS associé : `modules/hardware/xpt2046-hat.nix`
(option `services.navigation.hardware.hats.enableXpt2046`).

## Le matériel

Famille des écrans **Waveshare 3.5" 480x320** (et clones « waveshare35a ») : un
panneau **ILI9486** et un contrôleur tactile résistif **XPT2046** (compatible
**ADS7846**) qui partagent le bus **SPI0**.

- **SPI0.0** (CS0) : afficheur ILI9486 → framebuffer `fb_ili9486` → `/dev/fb1`.
- **SPI0.1** (CS1) : tactile ADS7846 → driver `ads7846` → `/dev/input/eventN`.
- **GPIO** : DC = **24**, RESET = **25**, PENIRQ tactile = **17**.

| Signal      | BCM GPIO |
| ----------- | -------- |
| SPI0 (bus)  | 7-11     |
| Display DC  | 24       |
| Display RST | 25       |
| Touch PENIRQ| 17       |

> Partage SPI0 + GPIO24/25 avec les HAT Pypilot et MacArthur → l'assertion de
> conflit refuse ces combinaisons (c'est voulu).

## Ce que fait le module

- Charge `spi-bcm2835`, `fbtft`, `fb_ili9486`, `ads7846`.
- Injecte un **overlay device-tree autonome** (pas de `.dtbo` vendeur requis) qui
  active SPI0, désactive les `spidev` par défaut et déclare l'afficheur (CS0) et
  le tactile (CS1), calqué sur `waveshare35a`.
- Options : `rotate` (0/90/180/270) et `spiFrequency` (Hz, défaut 16 MHz).
- Installe `tslib`, `evtest`, `xinput_calibrator`, `xinput`.

```nix
services.navigation.hardware.hats.enableXpt2046 = true;
services.navigation.xpt2046Hat = {
  rotate = 90;
  # spiFrequency = 16000000; # baisser si bruit/artefacts
};
```

## Bring-up et calibration au banc

```sh
# l'afficheur est-il là ?
ls -l /dev/fb1
con2fbmap 1 1            # bascule la console texte sur fb1 (si dispo)

# le tactile génère-t-il des événements ?
evtest                  # choisir le périphérique "ADS7846 Touchscreen"

# calibration console (tslib) -> /etc/pointercal
TSLIB_FBDEVICE=/dev/fb1 TSLIB_TSDEVICE=/dev/input/event0 ts_calibrate

# calibration X11 -> matrice à coller dans xorg.conf.d
xinput_calibrator
```

Console au démarrage sur `/dev/fb1` : ajouter `fbcon=map:10` (ou `map:1`) à la
cmdline du noyau. Sous X, pointer le serveur sur `fb1` (driver `fbdev`).

## À faire / vérifier au banc

Points non testables sans matériel, à confirmer (niveau 3) :

- **Bind du contrôleur** : `compatible = "ilitek,ili9486"` doit accrocher le
  driver `fb_ili9486` (fbtft). Selon le noyau (fbtft vs DRM tiny), le compatible
  ou le module peut différer ; vérifier `dmesg | grep -i 'ili9486\|fbtft'`.
- **Variantes de panneau** : les 2.x/3.2" utilisent un autre contrôleur
  (`fb_ili9341`), une autre résolution et parfois un PENIRQ différent. Ajouter des
  options de variante quand un de ces écrans sera testé.
- **Axes / rotation** : l'overlay pose `ti,swap-xy` et `rotate=0` par défaut ;
  l'orientation réelle et le sens du tactile sont à confirmer puis à figer via
  `rotate` + une matrice de calibration.
- **Persistance calibration** : `ts_calibrate` écrit `/etc/pointercal` et
  `xinput_calibrator` sort une matrice — prévoir de la rendre déclarative
  (fichier dans `/etc`) une fois mesurée plutôt qu'à refaire à chaque boot.
- **PENIRQ** : `interrupts = <17 2>` / `pendown-gpio = <&gpio 17 1>` supposent
  GPIO17 ; à vérifier sur la sérigraphie du HAT.
