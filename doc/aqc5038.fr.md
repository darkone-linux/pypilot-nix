# Kitronik 5038 — Air Quality Control HAT

Notes pour packager le driver, installer les dépendances et utiliser le HAT.
Module NixOS associé : `modules/hardware/aqc5038-hat.nix`
(option `services.navigation.hardware.hats.enableAqc5038`).

## Le matériel

HAT 40 broches (Pi 2/3/4/Zero/Zero 2). Deux canaux vers le Pi :

- **I2C-1** (GPIO2/3) :
  - **BME688** — capteur environnement + qualité d'air, adresse `0x77`
    (température, pression, humidité, résistance gaz, eCO2, indice AQ).
  - **OLED** 128x64 monochrome, contrôleur **SSD1306**, adresse `0x3C`.
- **serial0** (GPIO14/15, `/dev/serial0`, **115200 bauds**) :
  - co-processeur **RP2040** qui pilote 3 LED ZIP (WS2812), 3 entrées **ADC**
    (ADC0-2) et une **RTC**.

GPIO du header câblés par le HAT :

| Fonction        | BCM GPIO |
| --------------- | -------- |
| Buzzer piézo    | 26       |
| Sorties 1A      | 13, 19   |
| Servo           | 6        |
| Breakouts libres| 22,23,24 |

> Le RP2040 a besoin d'un `serial0` stable : le module pose `disable-bt` (déplace
> le Bluetooth hors de l'UART) et désactive le getty sur `ttyAMA0`.

## Packager pour nixpkgs (TODO)

### Driver

`pkgs/kitronik-air-quality-control-hat.nix`, `buildPythonPackage` depuis la sdist
PyPI ou le tag GitHub. Points d'attention :

- **`smbus`** : le driver fait `from smbus import SMBus`. nixpkgs ne fournit pas
  `python-smbus` seul ; deux options :
  - patcher le source en `from smbus2 import SMBus` (smbus2 est un drop-in et
    est déjà dans nixpkgs), **ou**
  - fournir `smbus-cffi` qui expose le module `smbus`.
- Autres deps déjà dans nixpkgs : `RPi.GPIO` (`python3Packages.rpi-gpio`),
  `pyserial`, `pillow`.
- `RPi.GPIO` ne tourne que sur Pi (accès `/dev/gpiomem`) → pas de test à l'import
  en sandbox ; `dontUseSetuptoolsCheck` / `pythonImportsCheck = []`.
- Pas de `setup.py` upstream propre : prévoir un `pyproject`/`format = "other"`
  minimal ou un `format = "pyproject"` avec un shim.

### luma-oled

`luma-core` est dans nixpkgs, **pas** `luma-oled`. À ajouter pour disposer d'une
lib SSD1306 générique (alternative au code OLED intégré du driver) :

- `buildPythonPackage` `luma.oled` depuis PyPI, deps `luma-core` + `pillow`.
- Utile pour du dessin OLED hors driver Kitronik.

Une fois les deux paquets faits : retirer le `pythonEnv` du module et exposer le
driver directement dans `environment.systemPackages`.

## Bring-up sur le banc

```sh
# bus I2C : BME688 à 0x77, OLED à 0x3c
i2cdetect -y 1

# le RP2040 répond sur serial0
ls -l /dev/serial0
```

Tant que le driver n'est pas packagé, environnement jetable :

```sh
python3 -m venv ~/aqc && . ~/aqc/bin/activate
pip install KitronikAirQualityControlHAT
```

Le module installe déjà `i2c-tools` et un `python3` avec `pyserial` + `smbus2`
pour le débogage direct des deux bus.
